cmake预定义的变量

  • PROJECT_SOURCE_DIR：工程根目录；

  • PROJECT_BINARY_DIR：运行cmake命令的目录。笔者建议定义为${PROJECT_SOURCE_DIR}/build下。具体原因见后文外部编译部分；

  • CMAKE_INCLUDE_PATH：环境变量，非cmake变量；

  • CMAKE_LIBRARY_PATH：环境变量；

  • CMAKE_CURRENT_SOURCE_DIR：当前处理的CMakeLists.txt文件所在路径；

  • CMAKE_CURRENT_BINARY_DIR：target编译目录；

    • 使用ADD_SURDIRECTORY指令可以更改该变量的值；
    • SET(EXECUTABLE_OUTPUT_PATH < dir >) 指令不会对该变量有影响，但改变了最终目标文件的存储路径；

  • CMAKE_CURRENT_LIST_FILE：输出调用该变量的CMakeLists.txt的完整路径；

  • CMAKE_CURRENT_LIST_LINE：输出该变量所在的行；

  • CMAKE_MODULE_PATH：定义自己的cmake模块所在路径；

  • EXECUTABLE_OUTPUT_PATH：重新定义目标二进制可执行文件的存放位置；

  • LIBRARY_OUTPUT_PATH：重新定义目标链接库文件的存放位置；

  • PROJECT_NAME：返回由PROJECT指令定义的项目名称；

  • CMAKE_ALLOW_LOOSE_LOOP_CONSTRUCTS：用来控制IF…ELSE…语句的书写方式；

  • CMAKE_C_FLAGS 设置C编译选项

  • CMAKE_CXX_FLAGS 设置C++编译选项

cmake_minimum_required(version 2.8)

指定最小的cmake版本

aux_source_directory

  • AUX_SOURCE_DIRECTORY(< dir > < variable >)
    • AUX_SOURCE_DIRECTORY(. SRC_LIST)
    • 将当前目录下的文件列表全部存入变量SRC_LIST中

add_library()导入库

添加一个库

target_link_libraries将若干库添加到目标文件

  • target_link_libraries( lib1 lib2 lib3)
  • 链接的顺序应当符合gcc链接顺序规则，被链接的库放在依赖它的库的后面，即如果上面的命令中，lib1依赖于lib2, lib2又依赖于lib3，则在上面命令中必须严格按照lib1 lib2 lib3的顺序排列，否则会报错

add_executable()添加可执行文件

  • add_executable( ${SRC_LIST})

set()设置cmake的变量，相当于为变量命名

  • SET(VAR [VALUE] [CACHE TYPEDOCSTRING [FORCE]])
  • 设置编译的类型
    • set(CMAKE_BUILD_TYPE DEBUG|RELEASE)
  • 设置编译选项
    • set(CMAKE_C_FLAGS_DEBUG “-g -Wall”)
    • set(CMAKE_CXX_FLAGS_DEBUG “-g -Wall”)
  • 指定最终二进制目标的位置
    • SET(EXECUTABLE_OUTPUT_PATH [out_path]) 设置执行文件的目标二进制位置
    • SET(LIBRARY_OUTPUT_PATH [out_path]) 设置库的最终二进制位置

include_directories()设置头文件的位置

link_directories添加需要链接的库文件目录

link_libraries添加需要链接的库文件的路径，注意这里是全路径

add_subdirectory()

该命令告诉CMake去子目录中查看可用的CMakeLists.txt文件
指令用于向当前工程添加存放源文件的子目录,并可以指定中间二进制和目标二进制存放的位置。 EXCLUDE_FROM_ALL 参数的含义是将这个目录从编译过程中排除。比如,工程的 example,可能就需要工程构建完成后,再进入 example 目录单独进行构建。
